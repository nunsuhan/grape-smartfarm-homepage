# 무손실 기술 블로그 편집 표준 (Lossless Technical Blog Editing Standard)

이 스킬은 FarmSense의 기술적 신뢰성을 담보하기 위해 NotebookLM 등에서 추출한 전문 자료를 웹 콘텐츠로 가공할 때 준수해야 할 편집 표준입니다. 정보의 단순화나 왜곡을 방지하고, 원문의 정밀함을 100% 유지하는 것을 목표로 합니다.

## 1. 수식 표기 표준 (Mathematical Formatting)
모든 수학적 공식, 화학식, 변수는 텍스트로 풀어쓰지 않고 LaTeX 문법을 사용하여 명확히 표기합니다.

- **규칙**: 인라인 수식은 `$` 사용, 블록 수식은 `$$` 사용.
- **Bad Examples**:
  - 증발산량은 ETc라고 씀.
  - y = 4.62x - 1.22 (일반 텍스트)
- **Good Examples**:
  - 증발산량($ET_c$)은...
  - $$ y = 4.62x - 1.22 $$ ($R^2=0.88$)

## 2. 상수 및 수치 데이터 보존 (Data Integrity)
논문 및 원문에 등장하는 모든 상숫값, 임계값, 범위 데이터는 절대 임의로 반올림하거나 버림 처리하지 않습니다.

- **규칙**: 소수점 단위까지 원문 그대로 유지.
- **예시**:
  - 포도 생육 임계 온도: "약 15도" (X) -> "14.9°C" (O)
  - 상관계수: "높음" (X) -> "$R^2 = 0.86$" (O)
  - 임계값: "20 이하" (X) -> "20.3°C 이하" (O)

## 3. 전문 용어 및 각주 활용 (Terminology & Footnotes)
독자의 이해를 돕기 위해 전문 용어를 쉬운 말로 대체하되, 괄호() 안에 반드시 원어(영어/학술명)를 병기하거나 각주를 통해 정확한 정의를 제공합니다.

- **규칙**: `쉬운 용어(전문 용어, Abbr)` 형태 유지. 필요 시 하단 각주 처리.
- **예시**:
  - "잎의 면적" (X) -> "엽면적지수(LAI, Leaf Area Index)" (O)
  - "물 부족 스트레스" (X) -> "수분 스트레스 지수(CWSI)" (O)
  - "적산온도" 뒤에 GDD(Growing Degree Days) 병기.

## 4. 구조화 및 위계 (Structure & Hierarchy)
정보의 위계를 명확히 하여 모바일에서도 가독성을 확보하되, 내용을 생략하지 않습니다.
- **요약문(tldr)**을 먼저 제시하고, 그 아래에 **상세 로직(Details)**을 배치하는 `Accordion`이나 `Toggle` UI 패턴 사용 권장.
